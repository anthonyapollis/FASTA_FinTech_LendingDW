Medallion ERD — This diagram shows the Gold layer star schema and key Silver operational tables. Bronze tables are the raw CSV sources. Silver adds cleansing and validation flags. Gold builds fact/dimension tables optimised for Power BI and Databricks SQL.
Gold Star Schema
Medallion Architecture — Three layers ensure data quality improves progressively. Databricks Delta Lake is the storage engine. MS Fabric would use OneLake with the same Delta format — the notebooks are compatible with both via spark.read / spark.write abstractions.

BRONZE — Raw Landing Zone

  • CSV/JSON files from source systems
  • No transformations applied
  • Append-only, never overwritten
  • 16 tables, 71,601 rows
  • Dirty data retained for audit
  • ADLS Gen2 / OneLake path: /mnt/landing/bronze/fasta/
  • Schema: auto-inferred or provided manifest
  • Partitioning: by ingestion date

SILVER — Cleansed & Validated

  • Deduplication (drop_duplicates on PK)
  • Date normalisation (6 format patterns)
  • Mixed case standardisation (UPPER/initcap)
  • Negative amount clipping → NULL
  • Future payment date flagging
  • Bot session removal (7% filtered)
  • DQ flags added: dq_missing_*, dq_future_*
  • Delta format, partitioned by date/brand
  • Accessible via silver.* Databricks catalog

GOLD — Analytics & Serving Layer

  • Star schema: 6 dims, 4 facts, 1 mart
  • dim_date, dim_customer, dim_brand, dim_risk_band
  • fact_loan_application — funnel KPIs
  • fact_disbursement — loan book
  • fact_repayment — collection performance
  • fact_web_funnel — digital acquisition
  • mart_channel_roi — channel quality ranking
  • Power BI DirectQuery via Databricks SQL
  • MS Fabric Lakehouse mirror via OneLake shortcut

Note on AUC: The credit risk AUC (0.47) is near-random because the Bronze layer data is synthetically generated — default labels are assigned randomly, not derived from actual credit behaviour. MLflow Tracking

All model runs logged with params, metrics, artifacts
# Databricks — MLflow auto-tracking
import mlflow
mlflow.set_experiment("/FASTA/CreditRisk")

with mlflow.start_run():
    model.fit(X_train, y_train)
    mlflow.log_metric("auc", auc)
    mlflow.log_param("n_estimators", 300)
    mlflow.sklearn.log_model(model, "credit_risk")

# Register in Unity Catalog
mlflow.register_model(
    "runs:/abc123/credit_risk",
    "fasta.models.credit_risk_v1"
)

# Score new applications in real-time
model = mlflow.sklearn.load_model(
    "models:/fasta.models.credit_risk_v1/Production"
)
scores = model.predict_proba(X_new)[:,1]

Feature Store (Delta)

Reusable features shared across models
# Databricks Feature Store
from databricks.feature_store import FeatureStoreClient

fs = FeatureStoreClient()

# Write customer credit features
fs.create_table(
    name="fasta.features.customer_credit",
    primary_keys=["customer_id"],
    df=customer_features_df,
    description="Credit & income features per customer"
)

# Use in training run
training_set = fs.create_training_set(
    df=loan_df,
    feature_lookups=[
        FeatureLookup(
            table_name="fasta.features.customer_credit",
            lookup_key="customer_id",
            feature_names=["credit_score","dsr_ratio","adverse_listings"]
        )
    ],
    label="default_label"
)
